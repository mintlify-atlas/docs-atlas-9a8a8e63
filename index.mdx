PA_PVP is a copy/paste decision protocol for AI systems. Give it a plan to pressure-test, an artifact to audit, or a choice to close — it returns a forced verdict (`DO NOW`, `DO LATER`, or `DISCARD`), one concrete next action, and a machine-readable snapshot you can feed back into the next run. No "it depends." No vague steps. Just an operational decision.

## What PA_PVP does

PA_PVP is a text-based execution contract — a system prompt you load into any LLM that supports instruction following. It applies a deterministic decision kernel to whatever you feed it.

For each item, the protocol:

1. Identifies the real decision or artifact under review
2. Exposes the highest-impact weakness or uncertainty
3. Assigns a verdict — `DO NOW`, `DO LATER`, or `DISCARD`
4. Produces executable or measurable next steps
5. Binds every claim to available evidence
6. Selects exactly one `[NEXT]` action
7. Preserves enough state for the next run

The result is a single machine-readable code block starting with `STATE: DECISION.TEST.FINAL`, followed by a human-readable table. Paste the entire output back as input to continue where you left off — no manual extraction required.

## Get started

<Steps>
  <Step title="Load the canonical protocol">
    Open [`PA_PVP_full_v9.9.0_canonical.txt`](https://github.com/xxyoudeadpunkxx/pa_pvp-a-plutonium-like-adversarial-peer-validation-protocol/blob/main/PA_PVP_full_v9.9.0_canonical.txt) and provide it to your AI system as a system prompt or context file.
  </Step>
  <Step title="Activate the protocol">
    Tell the AI to use PA_PVP as the active decision protocol:
    ```text
    Use PA_PVP as the active decision protocol for the next input. Follow its output contract exactly.
    ```
  </Step>
  <Step title="Describe your situation">
    Paste your plan, artifact, or decision in plain language. The protocol handles the rest.
    ```text
    This implementation plan looks plausible, but I'm not sure the dependencies and rollback path are strong enough. Use PA_PVP to pressure-test it and give me the next action.
    ```
  </Step>
  <Step title="Read the result and continue">
    The response contains a machine-readable code block and a human-readable table. To continue, paste the entire output back as input — no editing required.
  </Step>
</Steps>

<Note>
PA_PVP is a text protocol, not a software runtime. It structures decisions, simulations, evidence requests, probes, and next steps. Real execution depends on the AI host, available tools, and external systems.
</Note>
